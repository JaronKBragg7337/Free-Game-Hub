# Repository Verification Standard

A task is not complete after writing code.

Before reporting success, verify:

1. The commit exists.
2. The intended branch contains the commit.
3. The expected files exist in the repository.
4. The public route is known if GitHub Pages is used.

Every completion report should include:

- branch name,
- commit SHA if available,
- files changed,
- public URL if applicable,
- anything that could not be verified.

If verification fails, report the task as incomplete.
